Remote and Locale

Remote and locale are two important concepts when using Deploy-Man. Some components (as described in the next section) are remote, while other componets are locale.

A remote component (an Docker image or a configuration folder) is not on the locale machine (where this web interface runs), but on an Amazon S3 bucket. If you have such a component on your laptop, you need to upload it to S3 before you can use it. To upload a file you can use Deploy-Man or the AWS web console.

A locale component (a formation or a database setup) is always on the locale machine (where this web interface runs). Locale components doesn't need to be uploaded to Amazon, since they are only used inside Deploy-Man.

Basic componentes

Images

Images are Docker images saved as tarballs and uploaded to an Amazon S3 bucket. This means an image must always be in the S3 bucket. A local image must be uploaded to the S3 bucket first. Every image has a unique name by which it is indentifies. When a machine with a certain image is started, the image is copied to the machine, loaded into Docker and started.

An image usually goes together with a configuration.

Configs

Configs are folders with configurations (e.g. property files) uploaded to an Amazon S3 bucket. When a machine is started with a certain configuration, the configuration folder is copied to the machine and mounted into the Docker container specified in the formation file.
